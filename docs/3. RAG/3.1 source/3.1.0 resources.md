# Resources

This document organizes source resources by category so the RAG knowledge base can separate domain knowledge, technical standards, file formats, and parser test materials.

For now, this file only includes SAFF-related resources.


## 1. Structured Business File Formats

Structured business file formats describe how business data is exchanged between systems. These resources are useful for testing field extraction, table parsing, validation rule retrieval, and source-specific parser profiles.

### 1.1 SAFF, SuperStream Alternative File Format

SAFF is a SuperStream-related structured business file format used for exchanging superannuation contribution and member registration data.

Primary classification:

```text
Structured Business File Format
```

Related classifications:

```text
Payroll / Superannuation
SuperStream
Contribution reporting
Employer portal upload workflow
Parser test source
```

Recommended source metadata:

```text
source_type = "structured-business-file-format"
source_format = "pdf"
business_domain = "payroll-superannuation"
standard_family = "superstream"
format_name = "saff"
structure_type = "field-specification"
parser_profile = "pdf-table-aware-specification"
```

Parser notes:

1. Do not select the parser only from the file extension or MIME type.
2. SAFF resources are often published as PDFs, but the content may describe CSV/XLSX-like structured data.
3. Some SAFF PDFs are field specifications with dense tables.
4. Some SAFF PDFs are portal support guides with screenshots and step-by-step workflow instructions.
5. Use `structure_type` and `parser_profile` to choose the parsing strategy.

### ATO Data and Payment Standards - Contributions Message Implementation Guide

URL: https://softwaredevelopers.ato.gov.au/sites/default/files/resource-attachments/Schedule_4a_ContributionsMIG_v2.0_Apr2017.pdf

Version: 2.0

Release date: 20 April 2017

Applies from: 1 May 2017

Use for:

1. Official SuperStream contribution and member registration message requirements.
2. Standards context for SAFF contribution data mappings.
3. Business rules and data requirements that alternative formats should align with.
4. PDF parser tests for long standards documents with dense message content tables.

Suggested classification:

```text
source_type = "standards-implementation-guide"
source_format = "pdf"
business_domain = "payroll-superannuation"
standard_family = "superstream"
format_name = "saff"
structure_type = "message-implementation-guide"
parser_profile = "pdf-table-aware-standards-guide"
```

Notes:

1. This is an official ATO Schedule 4a resource for the Superannuation Data and Payment Standards.
2. The document does not directly define SAFF, but it defines the contribution and registration message requirements that SAFF-style alternative files should be mapped back to.
3. Section 12 covers alternate arrangements and requires alternative data mappings to show how alternative data elements map to the data elements in this guide.
4. Treat this as authoritative standards context for SAFF, not as a replacement for the ATO SAFF spreadsheet or provider-specific SAFF field specifications.
